# Web dashboard index

Description: Navigation map for the Vite/React marketing site, replay, observability dashboard, and SWE-bench evidence view.
Purpose: Identify UI ownership and the static/live data boundaries.

## Entry points and components

- `src/main.jsx` — mounts the React application.
- `src/App.jsx` — hash routing for the landing page, live token dashboard, and lazy-loaded benchmark view.
- `src/Landing.jsx` — marketing page, context timeline, and replay composition.
- `src/ContextTimeline.jsx` — self-contained four-state tool-call lifecycle illustration with autoplay and hover/focus selection.
- `src/ToolReplay.jsx` — self-contained, click-driven tool walkthrough using hand-authored context/check/return stories.
- `src/Dashboard.jsx` — live call timeline, original/intercepted comparison, and context-window overlay.
- `src/Benchmarks.jsx` — SWE-bench coding-performance summary and selectable paired-run ledger without a charting dependency.
- `src/data/benchmark-results.json` — mock snapshot-backed fixture or compact real artifact generated by `../benchmark/build_benchmark_results.py`.
- `src/data/demo-session.json` — unmounted frontend copy of the generated example session for development and future fixture use.
- `src/api.js` — fetches the current session and hydrates tool-call summaries with detail payloads.
- `src/useLiveSession.js` — polls the backend API for React consumers.
- `src/Antigravity.jsx` — React Three Fiber hero visualization.
- `public/ht6.svg` — Hack the 6ix logo displayed in the landing-page header.
- `public/gemini-svg.svg` — Loop-Whole browser favicon.
- `src/index.css` — shared application styling.
- `vite.config.js` — Vite configuration and development proxy to the gateway API.
- `eslint.config.js` — lightweight JavaScript and JSX lint configuration.
- `package.json` — frontend dependencies and `dev`, `lint`, `build`, and `preview` scripts.

## Backend integration boundary

- `ToolReplay` uses hand-authored static stories and runs independently of the backend; `Dashboard` consumes live gateway data through `src/useLiveSession.js`.
- `Benchmarks` consumes bundled paired data from `src/data/benchmark-results.json`, accepting mock nested snapshots or compact generated metrics; a top-level `mock` flag controls its warning eyebrow.
- `src/api.js` fetches the current snapshot and details, while `src/useLiveSession.js` polls it for React consumers.
- `../server/src/api.rs` defines `GET /health`, `GET /api/v1/sessions/current`, and `GET /api/v1/tool-calls/{id}`.
- `../server/src/schema.rs` is the authoritative camelCase response contract.
- The current-session endpoint returns lightweight call summaries. Fetch tool-call detail separately for original and intercepted payload text.
- The backend defaults to `127.0.0.1:8787`; browser integration needs either a Vite development proxy or backend CORS support.

No frontend test setup currently exists.
